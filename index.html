03

Exploring Commits & Branches

Understand the history of changes and different versions of the code.

üìú View Commit History

git log --oneline --graph --all

This shows a visual tree of all commits and branches!

üåø List All Branches

git branch -a

üîç See What Changed in a Commit

Replace <commit-hash> with an actual hash from git log

üîß Create a New Branch

05

Using Issues & Projects

Track tasks, bugs, and feature requests like a pro!

üé´ Creating an Issue

  • Go to the "Issues" tab on GitHub
  • Click "New Issue"
  • Give it a clear title and description
  • Add labels like "bug" or "enhancement"

üìã GitHub Projects

Organize issues with Kanban boards:

  • Create a project board
  • Add columns (To Do, In Progress, Done)
  • Link issues to track progress

üîó Linking Commits to Issues

git commit -m "Fix login bug (#42)"

Use #issue-number to automatically link commits!

06

GitHub Best Practices

Tips for working effectively with Git and GitHub.

‚úÖ Commit Message Guidelines

  • Use present tense: "Add feature" not "Added feature"
  • Be descriptive but concise
  • First line under 50 characters
  • Add detailed description after blank line if needed

üå≥ Branch Naming

  • feature/add-login for new features
  • fix/bug-navigation for bug fixes
  • docs/update-readme for documentation

üìù README Tips

  • Include project description
  • Add installation instructions
  • Provide usage examples
  • List prerequisites
  • Include contribution guidelines
